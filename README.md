## Що реалізовано

- **Жадібний алгоритм** — псевдокод п. 2.2.1. На кожному кроці обирає клітину і тип ліхтаря з найбільшим відношенням «нові освітлені клітини / вартість».
- **Алгоритм локального пошуку** — псевдокод п. 2.3.1. Стартує з розв'язку жадібного, покращує за 5 операторами: додавання, видалення, переміщення, зміна типу, заміна позиції.
- **Генератор задач** — п. 3.2. Чотири типи структур заборонених зон, фіксований seed для повторюваності.
- **Шість числових експериментів** — підрозділи 3.3.2–3.3.4 документа плюс три додаткові (бюджет, структура F, індивідуальні задачі).

### 1.1 Введення вручну

```
  Кількість рядків:    8
  Кількість стовпців:  8
  Вводьте координати заборонених клітин: рядок стовпець
```

## Суть задачі

Парк — прямокутна сітка m x n клітин.
Підмножина **F** — заборонені клітини (клумби, водойми, алеї).
Решта клітин **A = G \ F** — дозволені.

Мета: розмістити ліхтарі двох типів так, щоб **максимізувати кількість
освітлених дозволених клітин Q**.

### Два типи ліхтарів

| Тип | Зона покриття | Відстань Чебишева | Вартість |
|-----|--------------|-------------------|----------|
| 1 (стандартний) | 3 x 3 | не більше 1 | c1 |
| 2 (потужний) | 5 x 5 | не більше 2 | c2, де c2 > c1 |

Зона покриття обрізається по межах парку.
Заборонені клітини не рахуються в Q, навіть якщо потрапляють у зону покриття.
Освітлена дозволена клітина рахується **один раз**, незалежно від кількості ліхтарів.

### Обмеження задачі

- Загальна вартість ліхтарів не перевищує бюджет **B**
- Кількість ліхтарів типу 2 не перевищує **K2**
- В одній клітині не більше одного ліхтаря
- Ліхтарі встановлюються лише в дозволених клітинах (**A**)

Жадібний алгоритм виконує 7 кроків і досягає повного покриття Q = 58 = |A|:

|------|---------|-----|---------|----------|---|

## Використання компонентів у коді

### Розв'язати задачу

```python
from src.tasks import task_kalachuk
from src.algorithms import greedy, local_search

problem = task_kalachuk()

S_G, steps = greedy(problem, trace=True)
print(f"Жадібний: Q = {S_G.Q()} / {len(problem.A)}")

I_max = 4 * len(problem.A)
S_LS, hist = local_search(problem, S_G, I_max=I_max)
print(f"Локальний пошук: Q = {S_LS.Q()} / {len(problem.A)}")
```

### Згенерувати задачу

```python
from src.generator import generate

p = generate(
    m=12, n=12,
    rho_F=0.12,   # 12% заборонених клітин
    s_F=2,        # компактна структура (водойма)
    c1=3, c2=8,
    alpha_B=0.10, # B = round(0.10 * |A| * (c1+c2)/2)
    K2=3,
    seed=42,
)
print(p.info())
```

Параметр `s_F`: `1` — розсіяна, `2` — компактна, `3` — лінійна, `4` — два острови.
